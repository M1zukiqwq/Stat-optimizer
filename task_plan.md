## Decisions

| Decision | Rationale |
|---|---|
| Use additive experiment code where possible | Preserve existing paper/experiment contracts and avoid broad refactors. |
| Treat single-column-only multi-column estimation as maximum-entropy/independence | Exact joint stats are not identifiable from marginals alone; independence is the defensible no-extra-signal baseline. |
| Also evaluate a fixed-dependence copula path | Shows how corrected marginals help a real multi-column estimator when dependence metadata/model is already present. |
| Hybrid policy uses only feedback-window residuals | Avoids oracle/fresh-statistics leakage while testing whether deployment gating can prevent OASIS regressions. |
| Use PostgreSQL planner-only evidence instead of runtime | Avoids overclaiming end-to-end latency; directly tests whether corrected single-column statistics alter a real DBMS optimizer's estimates and plan choices. |
| Treat damped projection as experimental until downstream validation passes | It relaxes exact feedback matching, so it must be guarded by residual checks and validated in composition/FactorJoin/planner-facing settings before replacing full OASIS. |
| Keep hard OASIS-Proj as the planner/downstream default after soft projection validation | Soft projection improves cached single-column accuracy and optimizer-proxy regret, but it underperforms hard OASIS-Proj/Hybrid on FactorJoin, composition, OOD batch/range drift, DML trace append/cancellation, and PostgreSQL plan-shape safety. |
| Use temporal soft projection only as a calibrated variant, not a replacement | Recent-window soft (`window=8`, no recency decay inside the suffix) removes the worst full-window OOD/trace regressions and slightly improves PG plan-shape safety, but it loses some single-column/proxy gains and still trails OASIS-Proj/Hybrid in FactorJoin/composition row-QE. |
| Prefer conflict-aware soft over fixed recent8 when a soft Stage-2 is used | Gating feedback by consistency with the recent trusted reference (not by age) keeps old-but-consistent observations, so conflict-aware soft beats recent8 on single-column and optimizer-proxy accuracy at equal-or-lower feedback residual while still fixing full-window OOD/trace failures. It does not change the default: hard OASIS-Proj/Hybrid stays the planner/downstream anchor because soft still trails on FactorJoin/composition and strong OOD batch/range drift. |
| Evaluate Stage 2 across multiple Stage-1 prior sources | If Stage 2 becomes the paper's main story, the evidence should show that calibration is not merely an OASIS-MLP-specific patch; the formal swap should use the cached full test grid rather than a minimized smoke run. |

## Errors Encountered

| Error | Attempt | Resolution |
|---|---|---|
| Active-set soft projection too slow | Full expanded run with `--soft-projection-active-set` exceeded 3 minutes before completion; medium run also took about 2 minutes. | Kept active-set as an opt-in diagnostic, lowered its feasibility-check default to 50 iterations, and did not promote it. |
| Full-window recency-decay soft too slow | `--soft-projection-recency-decay 0.5` full-window runs consumed about 5 CPU minutes per task before normal progress output. | Stopped those runs and focused on cheap explicit recent-window soft projection. |
